## ui.json 规则（A2UI v0.9 协议）

- DynamicString（text, label, value, url）只用：纯字符串 或 { "path": "/..." }
- 绝对不要用 { "literalString": "..." }
- root 组件必须保留：id="root", component="Column"
- 新增的组件 id 必须符合 ^[a-z][a-z0-9-]*$（小写字母+数字+连字符）
- 所有 children 和 child 引用的 id 必须存在
- Button action.event.name 必须等于按钮 id
- Button variant 只能是 default / primary / borderless
- Text variant 只能是 h1 / h2 / h3 / h4 / h5 / body / caption
- TextField variant 只能是 shortText / longText / number / obscured
- Column / Row justify 只能是 start / center / end / spaceBetween / spaceAround / spaceEvenly / stretch
- Column / Row / List align 只能是 start / center / end / stretch
- Image fit 只能是 fill / contain / cover / none / scaleDown
- Image variant 只能是 icon / avatar / smallFeature / mediumFeature / largeFeature / header
- Card 只能有一个 child，多元素必须用 Column/Row 包装
- 新增 TextField 时：value 用 { "path": "/fieldId" } 路径绑定
- 新增 Button 时：action.event.context 包含所有需要传给 handler 的 TextField 路径引用

### 样式优化建议
- 用 justify/align 控制布局对齐：内容居中 → justify: "center"，两端分布 → justify: "spaceBetween"
- 用 weight 分配子元素空间比例（类似 CSS flex-grow）
- 主按钮用 variant: "primary"，次要操作用 "default"，轻量操作用 "borderless"
- 标题层次：h1 页面标题，h2/h3 区域标题，body 正文，caption 辅助说明
- 用 Divider 分隔不同内容区域

### 新增组件示例

添加第三个输入框：

同时需要在 root 的 children 中添加 "num3-input"，并在对应 Button 的 action.event.context 中添加 "num3": { "path": "/num3" }

## handlers.js 规则

- 函数名从 button id 转换：add-btn → onAddBtn, new-input-btn → onNewInputBtn
- 顶层（函数外）变量声明用 `var`，不要用 `const` 或 `let`（会被 STATE_LEAK 拒绝）
- 函数内任意 const/let/var 都可以
- 所有函数为同步函数（禁止 async/await/Promise）
- capability.ui.set/get 的第一个参数必须是字符串字面量（不是变量）
- capability.ui.set(id, property, value) 的 value 用纯字符串或数字
- 读取 TextField 用户输入：通过 __inputs__ 参数（需在 Button action.context 中配置 path 绑定）
- 不要用 capability.ui.get 读取 TextField 用户输入
- 禁止：eval, Function(), require, import, globalThis, fetch, XMLHttpRequest, Promise, WebSocket
- capability.ai.ask(prompt) → {status, body}，status===200 表示成功
- capability.network.fetch({url, ...}) → {status, body}，需在 capabilities 中声明 network 并带 allowlist
- 如果不知道真实 API 地址 → 不要用 network.fetch 编造 URL，改用 capability.ai.ask

### 修改现有函数时
- 保持原有函数名不变
- 如果新增了 TextField，记得在 __inputs__ 中读取新字段
- 如果新增了 Button，添加对应的 handler 函数

## 数据流提醒

TextField value: { "path": "/num1" }
       ↓ 用户输入
DataModel 自动更新
       ↓ 点击按钮
Button action context: { "num1": { "path": "/num1" } }
       ↓ A2UI 自动解析
Handler __inputs__: { "num1": "5" }

## 可用 API

- capability.storage.get(key) → string | null
- capability.storage.put(key, value) → void
- capability.storage.delete(key) → void
- capability.ui.set(id, property, value) → void（value 用纯字符串或数字）
- capability.ui.get(id, property) → string
- capability.os.notify(title, body) → void（需声明 os）
- capability.network.fetch({url, method?, headers?, body?}) → {status, body}（需声明 network 且带 allowlist）
- capability.ai.ask(prompt) → {status, body}（需声明 ai，用于 LLM 检索/搜索，不知道 API 地址时使用）

只输出三个文件，不要任何额外内容。
