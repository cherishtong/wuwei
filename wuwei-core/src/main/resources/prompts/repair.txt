## skill.json 验证规则

只有 7 个顶级字段：id, version, abi, runtime, meta, capabilities, signature
- id: 必须匹配 ^[a-z][a-z0-9-]*$（小写字母开头，只含小写字母、数字、连字符）
- version: 必须为 semver X.Y.Z 格式（如 1.0.0）
- abi: 必须固定为 "1.0"
- runtime: 必须是 "js" 或 "browser-js"
- capabilities: 只声明实际使用的
- 如果用了 capability.network.fetch → capabilities 必须有 `"network": {"allowlist": ["https://..."]}`，allowlist 不能为空
- 如果用了 capability.ai.ask → capabilities 必须有 `"ai": {}`
- 如果不知道真实 API 地址 → 不要用 network.fetch，改用 capability.ai.ask

## ui.json 验证规则（A2UI v0.9 协议）

- DynamicString（text, label, value, url）：只用纯字符串或 { "path": "/..." }，禁止 { "literalString": "..." }
- root 组件必须存在：id="root", component="Column"
- 所有 children 和 child 引用的 id 必须在 components 中存在
- Button 的 action.event.name 必须等于按钮的 id
- Button 的 variant 只能是：default / primary / borderless
- component 名只能是：Text / Button / TextField / Column / Row / Image / Icon / Divider / List / Card / Tabs / Modal / CheckBox / Slider / ChoicePicker / DateTimeInput / Switch / RadioGroup / Select / Toggle / ToggleGroup / InputOTP / Table / Badge / Progress / Avatar / Alert / Skeleton / AspectRatio / ScrollArea / Accordion / Collapsible / Carousel / Pagination / Breadcrumb / Video / AudioPlayer / Sheet / Drawer / AlertDialog / Tooltip / HoverCard / Popover / DropdownMenu / ContextMenu / Command / Canvas / Sonner
- Text 的 variant 只能是：h1 / h2 / h3 / h4 / h5 / body / caption
- Button 的 variant 只能是：default / primary / borderless
- TextField 的 variant 只能是：shortText / longText / number / obscured
- Image 的 fit 只能是：fill / contain / cover / none / scaleDown
- Image 的 variant 只能是：icon / avatar / smallFeature / mediumFeature / largeFeature / header
- Column / Row 的 justify 只能是：start / center / end / spaceBetween / spaceAround / spaceEvenly / stretch
- Column / Row / List 的 align 只能是：start / center / end / stretch
- Divider 的 axis 只能是：horizontal / vertical
- List 的 direction 只能是：vertical / horizontal
- TextField value 如果需要在 handler 中读取，必须用 { "path": "/..." } 路径绑定
- Button action.event.context 中传入的 path 引用必须对应真实存在的 TextField
- Card 的 child 引用必须存在，且 Card 只能有一个 child

## handlers.js 验证规则

**变量声明规则（重要）：**
- 顶层（函数外）：用 `var` 声明状态变量（`let` 和 `const` 在顶层会被拒绝）
- 函数内：`const` / `let` / `var` 都可以使用
- 正确示例：顶层 `var counter = 0;`  ← 允许
- 错误示例：顶层 `let counter = 0;` 或 `const counter = 0;` ← 被拒绝（STATE_LEAK）

**其他规则：**
- 函数名必须从 button id 转换：add-btn → onAddBtn, query-btn → onQueryBtn
- capability.ui.set/get 的第一个参数必须是字符串字面量（不是变量），且该 id 必须在 ui.json 中存在
- capability.ui.set(id, property, value) 的 value 用纯字符串或数字
- 禁止：eval, Function(), require, import, globalThis, fetch, XMLHttpRequest, Promise, WebSocket, Worker
- 禁止 async/await/Promise
- 读取 TextField 用户输入的正确方式：__inputs__.fieldName（来自 Button action.context 的 path 绑定）
- 不要用 capability.ui.get 读取 TextField 用户输入
- capability.ai.ask(prompt) 返回 {status, body}，status===200 表示成功
- capability.network.fetch 需要 capabilities 中有 network 且带 allowlist
- 不知道真实 API 地址时不要编造 URL，改用 capability.ai.ask

## 常见错误及具体修复方法

### INVALID_MANIFEST
- id "CounterTool" → "counter-tool"（kebab-case）
- id "counter_tool" → "counter-tool"（不用下划线）
- id "-counter" → "counter"（不以连字符开头）
- id "123tool" → "tool-123"（不以数字开头）
- version "1.0" → "1.0.0"（需要补丁版本号）
- abi "1" → "1.0"（需要 X.Y 格式）
- 有 8 个顶级字段 → 删掉多余字段，只保留 7 个

### UNDEFINED_ID
- capability.ui.set("result", ...) 但 ui.json 中 id 是 "result-text" → 改为 "result-text"
- children 中有 "submit-btn" 但 components 中无此 id → 添加该组件或删掉引用

### DANGLING_CHILD_REF
- Button child 引用了 "btn-text" 但该组件不存在 → 添加对应的 Text 组件
- children 包含不存在的 id → 检查拼写，确保引用的组件已定义

### MISSING_ROOT
- components 中没有 id="root" 的 Column → 添加 { "id": "root", "component": "Column", "children": [...] }

### BAD_COMPONENT_NAME
- "component": "Container" → "component": "Column"（不是有效组件名）
- "component": "Input" → "component": "TextField"

### BAD_BUTTON_VARIANT
- variant "blue" → "primary"
- variant "outline" → "default"

### BAD_TEXT_VARIANT
- variant "title" → "h1"
- variant "subtitle" → "h2"
- variant "large" → "h1"
- variant "p" → "body"
- variant "small" → "caption"

### BAD_JUSTIFY
- justify "left" → "start"
- justify "right" → "end"
- justify "space-between" → "spaceBetween"
- justify "space-around" → "spaceAround"

### BAD_ALIGN
- align "top" / "left" → "start"
- align "bottom" / "right" → "end"
- align "middle" → "center"
- align "stretched" → "stretch"

### BAD_IMAGE_VARIANT
- variant "thumb" → "smallFeature"
- variant "full" → "largeFeature"
- variant "profile" → "avatar"

### BAD_IMAGE_FIT
- fit "stretch" → "cover"
- fit "scale-down" → "scaleDown"

### CARD_MULTIPLE_CHILDREN
- Card 的 children 数组有多个元素 → 改为单个 child，用 Column/Row 包装多个元素

### STATE_LEAK
- 顶层 const x = 0; → var x = 0;（顶层只能用 var）
- 顶层 let y = []; → var y = [];（或移到函数内部）

### CAPABILITY_ESCAPE
- 使用了 capability.storage 但 skill.json 中没声明 → 在 capabilities 中添加 "storage": {}，或者删掉 storage 相关代码
- 使用了 capability.network 但没声明 → 同上

### DYNAMIC_UI_ID
- capability.ui.set(idVar, ...) → capability.ui.set("具体的id", ...)
- 第一个参数不能是变量/表达式，必须是字符串字面量

### SYNTAX_ERROR
- 缺少括号/引号不匹配 → 检查语法
- 函数体末尾缺少 } → 补全
- T.AmbientLight(0x) / T.Color(0x) 等缺失 hex 数字 → 补全为有效的 6 位 hex 颜色，如 0xffffff 或 0x404040
- 0x 后面必须有 6 位十六进制数字（0-9, a-f）

### LITERAL_STRING_WRAPPER
- { "text": { "literalString": "hello" } } → { "text": "hello" }

### MISSING_PATH_BINDING
- TextField value 是纯字符串但 handler 需要读取 → 改为 { "path": "/fieldId" }

### MISSING_ACTION_CONTEXT
- Button 有点击事件但 handler 需要读取 TextField 值 → 添加 action.event.context

### FAKE_API_URL
- network.fetch 使用了编造的 URL（如 api.example.com），但不知道真实 API → 改用 capability.ai.ask

### MISSING_NETWORK_ALLOWLIST
- 用了 network.fetch 但 capabilities 中 network 没有 allowlist → 添加 allowlist 前缀

### MISSING_AI_CAPABILITY
- 用了 capability.ai.ask 但 capabilities 中没有声明 ai → 添加 "ai": {}

### MISSING_CANVAS_CAPABILITY
- 用了 capability.canvas.render 但 capabilities 中没有声明 canvas → 添加 "canvas": {}
- ui.json 中有 Canvas 组件但没声明 canvas capability → 添加声明

### MISSING_THREEJS_CAPABILITY
- 用了 capability.threejs 但 capabilities 中没有声明 threejs → 添加 "threejs": {}
- 用了 threejs 但 runtime 不是 "browser-js" → 改为 "runtime": "browser-js"

### RUNTIME_MISMATCH
- 使用了 async/await 但 runtime 是 "js" → 改为 "runtime": "browser-js"（GraalJS 不支持 async）
- 使用了 capability.threejs 但 runtime 是 "js" → 必须用 "browser-js"
- 使用了 capability.canvas 但 runtime 是 "js" → 必须用 "browser-js"
- js 运行时顶层用了 let/var → 改为 const（js 运行时只允许 const 顶层声明）

### BROWSER_JS_VAR
- browser-js 运行时顶层用了 const/let 声明状态 → 改为 var（browser-js 中只用 var 声明持久状态）
- 这不是错误，但如果函数内用 var 声明也是正确的

### ASYNC_CAPABILITY_MISSING_AWAIT
- browser-js 中调用了 capability.storage.get() 但没有 await → 添加 await
- browser-js 中调用了 capability.ai.ask() 但没有 await → 添加 await
- browser-js 中调用了 capability.threejs.init() 但没有 await → 添加 await

### THREEJS_NEW_KEYWORD
- 使用了 `new THREE.BoxGeometry(...)` → 改为 `T.BoxGeometry(...)`（不需要 new）
- 属性设置用了 `.position.set(x,y,z)` → 改为 `.position.x = x; .position.y = y; .position.z = z`

### CARD_INVALID_CHILD
- Card 的 child 引用了不存在的组件 → 确保 child 指向已定义的组件
- Card 使用了 children（数组）→ 改为 child（单个组件 id）

### BAD_SEVERITY
- Alert 的 severity 不在允许值中 → 改为 info / warning / error / success

### BAD_TABLE_COLUMNS
- Table 缺少 columns 或 rows 属性 → 必须同时提供
- columns 中缺少 title 或 key → 补充完整

### DB_SQL_INJECTION
- SQL 语句包含多个分号分隔的语句 → 改为单语句
- SQL 使用了禁止的关键词（PRAGMA、ATTACH、DETACH、VACUUM）→ 移除这些语句，用标准 SQL 代替
- SQL 不是以允许的 DDL（CREATE TABLE/INDEX/VIEW, ALTER TABLE, DROP TABLE/INDEX）或 DML（SELECT/INSERT/UPDATE/DELETE/REPLACE）开头 → 修正 SQL 语句
- 注意：capability.db.run() 仅允许 DDL 单语句，capability.db.query() 仅允许 SELECT，capability.db.execute() 仅允许 INSERT/UPDATE/DELETE/REPLACE

### DB_SIZE_LIMIT
- 数据库文件超过 100MB 限制 → 减少数据量，删除不必要的记录，或优化表结构
- 如果需要存储大量数据，考虑用 capability.file 代替

### DB_QUERY_TIMEOUT
- SQL 查询超过 5 秒超时 → 添加索引、优化查询条件、减少返回行数
- 对于复杂查询，拆分为多次简单查询

### CRYPTO_PARAM_TOO_LARGE
- capability.crypto.randomBytes(n) 的 n 超过 1024 → 改为 1024 或更小的值
- 注意：randomBytes 上限为 1024 字节

### MODULE_NOT_FOUND
- require('./xxx') 路径找不到对应的 .js 文件 → 检查文件路径是否正确，确保模块文件存在于 handlers/ 目录下
- 路径不能以 / 开头，不能包含 ..（路径穿越禁止）
- 确保模块文件名以 .js 结尾（可省略，自动补全）

### CIRCULAR_REF
- UI $ref 中存在循环引用 → 检查 ui/ 中的 $ref 链，打破循环
- 例如 A → B → A 的引用链是不允许的

### MISSING_DATABASE_CAPABILITY
- 使用了 capability.db.run/query/execute/transaction 但 capabilities 中没有声明 database → 添加 "database": {}

### MISSING_WEBSEARCH_CAPABILITY
- 使用了 capability.websearch.search 但 capabilities 中没有声明 websearch → 添加 "websearch": {}

### MISSING_CRYPTO_CAPABILITY
- 使用了 capability.crypto.* 但 capabilities 中没有声明 crypto → 添加 "crypto": {}
